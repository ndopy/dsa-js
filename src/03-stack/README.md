# 스택 (Stack)

## 개념
- 데이터를 차곡차곡 쌓아 올린 형태의 자료구조이다.
- 가장 마지막에 삽입된 데이터가 가장 먼저 삭제되는 **후입선출(LIFO, Last In First Out)** 구조를 가진다.
- 이 구현체에서는 이전에 구현한 **이중 연결 리스트(Doubly Linked List)** 를 내부 저장소로 활용하여 구현하였다.

<br/>

## 추상자료형 ADT
| 함수 시그니처 | 시간 복잡도 | 설명 |
| --- | --- | --- |
| `push(data)` | $O(1)$ | 스택의 맨 위(Top)에 데이터를 추가한다. |
| `pop()` | $O(1)$ | 스택의 맨 위(Top)에 있는 데이터를 꺼내고(삭제하고) 반환한다. |
| `peek()` | $O(1)$ | 스택의 맨 위(Top)에 있는 데이터를 반환하되, 삭제하지는 않는다. |
| `isEmpty()` | $O(1)$ | 스택이 비어있는지 여부를 반환한다. |
| `getSize()` | $O(1)$ | 현재 스택에 저장된 데이터의 개수를 반환한다. |

<br/>

## 핵심 로직

### 연결 리스트 활용 (Adapter Pattern)
- 데이터를 직접 관리하는 대신, `DoublyLinkedList` 인스턴스를 생성하여 내부 저장소로 사용한다.
- 스택의 기능(`push`, `pop`)을 연결 리스트의 메서드(`insertAtBack`, `removeBack`)로 연결(위임)하여 구현한다.

### 방향성 설정 (Top의 위치)
- 리스트의 **가장 뒤(Tail)** 를 스택의 **맨 위(Top)** 로 간주한다.
- 따라서 데이터를 넣을 때도 뒤로 넣고(`insertAtBack`), 뺄 때도 뒤에서 뺀다(`removeBack`).
- 입구와 출구가 같은 방향이어야 스택의 LIFO 특성이 유지된다.

<br/>

## 주의사항

### 빈 스택에서의 연산
- 스택이 비어있을 때 `pop()`이나 `peek()`을 호출하면 에러가 발생하거나 `null`/`undefined`가 반환될 수 있다.
- 내부 연결 리스트의 `removeBack()` 구현에 따라 빈 리스트에서의 삭제 시 에러 처리가 결정된다.
