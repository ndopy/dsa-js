# 메모이제이션 (Memoization)

## 1. 개념
컴퓨터 프로그램이 동일한 계산을 반복해야 할 때, 이전에 계산한 값을 메모리에 저장(캐싱)해 둠으로써 중복 계산을 제거하여 전체적인 실행 속도를 빠르게 하는 기술이다. 동적 계획법(Dynamic Programming)의 **하향식(Top-down)** 접근 방식에서 주로 사용된다.

## 2. 핵심 특징
1. **캐싱 (Caching):** 한 번 계산된 결과는 별도의 메모리 공간(객체, 배열 등)에 저장해 둔다.
2. **재사용:** 이후 동일한 입력으로 함수가 호출되면, 다시 계산하지 않고 저장된 값을 즉시 반환한다.
3. **성능 최적화:** 피보나치 수열과 같이 중복 호출이 많은 재귀 함수의 시간 복잡도를 지수 시간($O(2^n)$)에서 선형 시간($O(n)$)으로 획기적으로 줄여준다.

## 3. 구현 방식 (Top-down)
1. 재귀 함수를 기본 구조로 사용한다.
2. 함수 진입 시, 해당 입력값에 대한 결과가 메모리에 있는지 확인한다.
3. 있다면 저장된 값을 반환하고, 없다면 계산을 수행한 뒤 결과를 저장하고 반환한다.

## 4. 구현 팁 (안전한 조건 검사)
- 자바스크립트에서 `0`, `null`, `undefined` 등은 모두 **Falsy** 값으로 취급된다.
- 만약 계산 결과가 `0`일 수 있는 경우, `if (!memo[n])`과 같이 검사하면 이미 계산된 `0`을 "값이 없다"고 착각하여 재계산을 수행하게 된다.
- **해결책:** 값이 명확하게 존재하지 않음을 확인하기 위해 `memo[n] === undefined`와 같이 엄격하게 비교하는 것이 안전하다.

```javascript
// 권장 방식
if (memo[n] === undefined) { ... }
```

- 피보나치 수열 (Fibonacci Sequence)
