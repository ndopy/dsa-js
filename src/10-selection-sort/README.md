# 선택 정렬 (Selection Sort)

## 1. 개념
배열을 순회하며 정렬되지 않은 부분 중에서 **가장 작은 값(또는 가장 큰 값)** 을 찾아, 해당 부분의 **맨 앞 위치와 교환(Swap)** 하는 방식으로 정렬하는 알고리즘이다. <br/>
데이터 중에서 순서에 맞는 값을 '선택'하여 위치를 확정한다고 하여 선택 정렬이라 부른다.

## 2. 핵심 특징
1. **단순함:** 구현이 매우 직관적이고 이해하기 쉽다.
2. **제자리 정렬 (In-place Sorting):** 데이터를 저장하는 배열 외에 추가적인 메모리 공간이 거의 필요 없다.
3. **불안정 정렬 (Unstable Sort):** 값이 같은 요소들의 상대적인 순서가 유지되지 않을 수 있다. (예: `[5, 5, 1]` 정렬 시 첫 번째 5가 1과 교환되어 뒤로 이동함)
4. **교환 횟수 최소화:** 비교 횟수는 많지만, 실제로 값을 교환(Swap)하는 횟수는 버블 정렬에 비해 현저히 적다.

## 3. 시간 복잡도 (Time Complexity)
1. **최악, 평균, 최선:** O(n²) - 정렬 여부와 관계없이 항상 나머지 모든 요소를 비교하여 최솟값을 찾아야 하므로 시간 복잡도가 일정하다.

## 4. 구현 팁 (성능 최적화)
- **불필요한 교환 방지:** 탐색이 끝난 후 찾은 최솟값의 위치(`minIndex`)가 현재 위치(`i`)와 같다면, 이미 제자리에 있는 것이므로 교환(Swap) 연산을 건너뛴다.
