# 버블 정렬 (Bubble Sort)

## 1. 개념
서로 인접한 두 원소를 비교하여, 정렬 기준(오름차순 등)에 맞지 않으면 위치를 교환(Swap)하며 정렬하는 알고리즘이다. <br/>
마치 거품(Bubble)이 수면 위로 올라오는 듯한 모습을 보인다고 하여 버블 정렬이라 부른다.

## 2. 핵심 특징
1. **단순함:** 구현이 매우 직관적이고 이해하기 쉽다.
2. **제자리 정렬 (In-place Sorting):** 데이터를 저장하는 배열 외에 추가적인 메모리 공간이 거의 필요 없다.
3. **안정 정렬 (Stable Sort):** 값이 같은 요소들의 상대적인 순서가 유지된다.
4. **비효율성:** 데이터의 개수가 많아질수록 성능이 급격히 저하된다.

## 3. 시간 복잡도 (Time Complexity)
1. **최악 및 평균:** O(n²) - 이중 반복문을 사용하여 모든 요소를 비교할 때 발생한다.
2. **최선:** O(n) - 이미 정렬되어 있는 경우 (단, 조기 종료 최적화가 적용된 경우).

## 4. 구현 팁 (성능 최적화)
- **조기 종료 (Early Exit):** 한 회전(Pass) 동안 데이터의 교환(Swap)이 한 번도 일어나지 않았다면, 이미 정렬이 완료된 상태이므로 반복을 즉시 종료한다.
- **범위 축소:** 각 회전이 끝날 때마다 가장 큰 값(또는 작은 값)이 맨 뒤로 이동하여 자리를 잡으므로, 다음 회전부터는 비교 범위를 1씩 줄여나간다.
