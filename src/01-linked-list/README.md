# 연결 리스트 (Linked List)

## 개념
데이터를 **노드(Node)** 라는 단위로 나누어 메모리상에 비연속적으로 저장하는 방식을 사용하는 자료구조이다.

<br/>

## 추상자료형 ADT
| 함수 시그니처 | 시간 복잡도 | 설명 |
| --- | --- | --- |
| `printAll()` | O(n) | 리스트 안에 있는 모든 데이터를 순서대로 콘솔에 출력한다. |
| `clear()` | O(1) | 리스트를 완전히 비워 초기 상태로 만든다. |
| `insertAt(index, data)` | O(n) | 리스트의 특정 위치(index)에 새로운 데이터를 삽입한다. |
| `insertLast(data)` | O(n) | 리스트의 가장 마지막 부분에 새로운 데이터를 삽입한다. |
| `deleteAt(index)` | O(n) | 특정 위치(index)에 있는 데이터를 삭제하고, 삭제된 노드를 반환한다. |
| `deleteLast()` | O(n) | 리스트의 가장 마지막에 있는 데이터를 삭제하고, 삭제된 노드를 반환한다. |
| `getNodeAt(index)` | O(n) | 특정 위치(index)에 있는 노드(Node) 객체를 찾아 반환한다. |

<br/>

## 핵심 로직

### 노드의 연결
- 각 노드는 `data`와 다음 노드를 가리키는 `next`를 가진 객체로 이루어진다. <br/> 이 `next`를 이용해 데이터들이 한 줄로 이어지게 된다.

### 삽입 로직
- 새로운 노드를 중간에 넣을 때, 이전 노드의 `next`를 새 노드에 연결하고, <br/>새 노드의 `next`를 원래 다음 순서였던 노드에 연결하는 과정이 핵심이다.

### 삭제 로직
- 특정 노드를 삭제하려면 이전 노드가 삭제할 노드의 다음 노드를 바로 가리키도록 연결을 수정한다.
- 이렇게 하면 삭제 대상인 노드는 연결에서 떨어져 나가게 된다.

<br/>

## 주의사항

### 인덱스 범위 확인
- 데이터를 삽입하거나 삭제할 때 <br/> 인덱스가 0보다 작거나 현재 데이터 개수보다 큰 경우를 반드시 확인하여 오류를 방지해야 한다.

### 반복문 범위
- 특정 인덱스의 이전 노드까지만 이동해야 하는 경우가 많으므로, <br/> 반복문의 횟수 설정(`index - 1` 등)을 정확하게 작성해야 한다.

### 연결 순서
- 삽입 / 삭제 시, 연결 순서를 잘못 바꾸면 뒤쪽 노드들에 대한 정보가 사라져 버릴 수 있으므로, <br/> 연결 순서를 논리적으로 잘 지켜야 한다.
