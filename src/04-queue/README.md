# 큐 (Queue)

## 개념
- 줄을 서는 것과 같이 먼저 들어온 데이터가 먼저 처리되는 형태의 자료구조이다.
- 가장 먼저 삽입된 데이터가 가장 먼저 삭제되는 **선입선출(FIFO, First In First Out)** 구조를 가진다.
- 이 구현체에서는 이전에 구현한 **이중 연결 리스트(Doubly Linked List)** 를 내부 저장소로 활용하여 구현하였다.

<br/>

## 추상자료형 ADT
| 프로퍼티 / 메서드 | 시간 복잡도 | 설명 |
| --- | --- | --- |
| `enqueue(data)` | $O(1)$ | 큐의 가장 뒤(Rear)에 데이터를 추가한다. |
| `dequeue()` | $O(1)$ | 큐의 가장 앞(Front)에 있는 데이터를 꺼내고(삭제하고) 반환한다. |
| `peek()` | $O(1)$ | 큐의 가장 앞(Front)에 있는 데이터를 반환하되, 삭제하지는 않는다. |
| `isEmpty()` | $O(1)$ | 큐가 비어있는지 여부를 반환한다. |
| `clear()` | - | 큐의 모든 데이터를 삭제하여 초기화한다. |
| `size` | $O(1)$ | 현재 큐에 저장된 데이터의 개수를 반환한다. (Getter로 구현) |

<br/>

## 핵심 로직

### 연결 리스트 활용 (Adapter Pattern)
- 데이터를 직접 관리하는 대신, `DoublyLinkedList` 인스턴스를 생성하여 내부 저장소로 사용한다.
- 큐의 기능(`enqueue`, `dequeue`)을 연결 리스트의 메서드(`insertAtBack`, `removeFront`)로 연결(위임)하여 구현한다.

### 방향성 설정 (Front와 Rear)
- 리스트의 **가장 앞(Head)** 을 큐의 **맨 앞(Front)**, **가장 뒤(Tail)** 를 큐의 **맨 뒤(Rear)** 로 간주한다.
- 데이터는 뒤로 들어가고(`insertAtBack`), 앞에서 나온다(`removeFront`).
- 입구와 출구가 반대 방향이어야 큐의 FIFO 특성이 유지된다.

<br/>

## 주의사항

### 빈 큐에서의 연산
- `peek()`: 큐가 비어있을 경우 `null`을 반환하도록 안전하게 처리한다.
- `dequeue()`: 내부 연결 리스트의 `removeFront()`를 호출하므로, 빈 큐에서 호출 시 에러가 발생한다.
