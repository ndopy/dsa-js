# 재귀 함수 (Recursion)

## 1. 개념
함수 내부에서 자기 자신을 다시 호출하여 작업을 수행하는 프로그래밍 기법이다.

## 2. 핵심 특징
1. **접근 방식:** 반복문(for loop)이 주로 상향식(Bottom-up) 처리에 사용된다면, 재귀 함수는 큰 문제를 작은 문제로 쪼개는 **하향식(Top-down)** 접근에서 강점을 발휘한다.
2. **기저 조건 (Base Case):** 무한 루프에 빠지지 않기 위해 재귀 호출을 멈추는 탈출 조건이 **반드시** 필요하다.
3. **콜 스택 (Call Stack):** 기저 조건을 작성하지 않거나 호출 깊이가 너무 깊어지면, 콜 스택이 한계치까지 쌓여 **스택 오버플로우(Stack Overflow)** 에러가 발생한다.

## 3. 재귀 함수의 2가지 패턴
1. **단순 반복:** 작업을 단순히 반복 수행하기 위해 사용한다.
2. **하위 문제 해결:** 하위 문제의 결과를 반환받아 상위 문제를 해결하는 방식이다. (점화식 형태)

## 4. 구현 팁 (성능 최적화)
- 배열이나 문자열을 다룰 때 매번 `slice` 등으로 자르면 메모리 낭비가 발생할 수 있다.
- 대신 **인덱스(index)** 를 인자로 전달하여 원본 데이터를 유지한 채 탐색하는 방식이 효율적이다.

## 5. 실습 예제
- 배열의 합 구하기
- 문자열 길이 구하기
