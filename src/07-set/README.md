# 해시 셋 (HashSet)

## 개념
- **HashSet**은 중복되지 않는 유일한 데이터들의 집합을 관리하는 자료구조이다.
- 데이터의 순서를 보장하지 않는다.
- 내부적으로 **해시 테이블(Hash Table)** 을 사용하여 구현되어 있어, 데이터의 검색, 삽입, 삭제가 매우 빠르다.

<br/>

## 추상자료형 ADT
| 프로퍼티 / 메서드 | 시간 복잡도 | 설명 |
| --- | --- | --- |
| `add(data)` | $O(1)$ | 셋에 데이터를 추가한다. 이미 존재하는 데이터라면 추가하지 않는다. |
| `remove(data)` | $O(1)$ | 셋에서 특정 데이터를 삭제한다. 성공 시 `true`, 실패 시 `false`를 반환한다. |
| `contains(data)` | $O(1)$ | 셋에 특정 데이터가 존재하는지 확인한다. |
| `clear()` | - | 셋의 모든 데이터를 삭제하여 초기화한다. |
| `isEmpty()` | $O(1)$ | 셋이 비어있는지 여부를 반환한다. |
| `size` | $O(1)$ | 현재 셋에 저장된 데이터의 개수를 반환한다. (Getter로 구현) |

*시간 복잡도는 해시 충돌이 거의 없는 평균적인 경우를 가정한다.*

<br/>

## 핵심 로직

### 해시 테이블 활용 (Composition)
- `HashSet`은 데이터를 직접 관리하지 않고, 내부적으로 `HashTable` 인스턴스를 생성하여 데이터 저장을 위임한다.
- **Key-Value** 구조인 해시 테이블을 **Key**만 필요한 셋(Set)으로 활용하기 위해, Value에는 의미 없는 고정값(Dummy Value)을 저장한다.

### 데이터 유일성 보장
- 해시 테이블의 Key는 중복될 수 없다는 특성을 이용하여 자연스럽게 집합의 유일성(Unique) 조건을 만족시킨다.
- `add()` 호출 시, 해시 테이블의 `put()` 메서드를 통해 데이터를 저장하며, 이때 Value로는 `true` (또는 특정 상수)를 사용한다.

<br/>

## 주의사항

### 데이터 타입
- 현재 구현된 해시 테이블의 해시 함수(`charCodeAt` 사용) 특성상, **문자열(String)** 데이터만 정상적으로 처리할 수 있다.
