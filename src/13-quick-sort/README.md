# 퀵 정렬 (Quick Sort)

## 1. 개념
기준값(Pivot)을 하나 선정하고, 피벗보다 작은 값은 왼쪽으로, 큰 값은 오른쪽으로 분할(Partition)한 뒤, 나누어진 부분 리스트를 재귀적으로 정렬하는 알고리즘이다.<br/>
병합 정렬과 마찬가지로 **분할 정복(Divide and Conquer)** 알고리즘에 속한다.

## 2. 핵심 특징
1. **분할 정복:** 문제를 작은 문제(피벗 기준 좌우 분할)로 쪼개서 해결한다.
2. **불안정 정렬 (Unstable Sort):** 값이 같은 요소들의 상대적인 순서가 바뀔 수 있다.
3. **제자리 정렬 (In-place Sorting):** 병합 정렬과 달리 별도의 추가 배열을 사용하지 않고, 입력 배열 내부에서 교환을 통해 정렬을 수행한다. (단, 재귀 호출로 인한 스택 메모리는 사용된다.)
4. **빠른 속도:** 평균적으로 매우 빠르며, 데이터의 지역성(Locality)이 좋아 캐시 적중률(Cache Hit Rate)이 높다.

## 3. 시간 복잡도 (Time Complexity)
1. **평균, 최선:** O(n log n)
   - 피벗이 리스트를 균등하게 나눌 때 가장 효율적이다.
2. **최악:** O(n^2)
   - 이미 정렬된 배열이나 역순 배열에서 피벗을 최솟값이나 최댓값으로 잡는 경우, 분할이 불균형하게 일어나 성능이 저하된다.
   - 하지만 피벗 선택 전략을 통해 이러한 최악의 경우를 방지할 수 있어 실무에서 가장 많이 사용된다.

## 4. 구현 팁 (성능 최적화)
- **피벗 선택 (Pivot Selection):** 
  - 단순히 첫 번째나 마지막 요소를 피벗으로 잡으면 정렬된 데이터에서 최악의 성능(O(n^2))을 보인다. 
  - 이를 방지하기 위해 **중간값(Median of Three)** 을 피벗으로 선택하거나, **무작위(Random)** 로 선택하는 방법을 사용한다.
- **작은 부분 배열:** 데이터의 개수가 적을 때(보통 10~20개 이하)는 퀵 정렬의 재귀 오버헤드보다 **삽입 정렬(Insertion Sort)** 이 더 빠를 수 있으므로, 하이브리드 방식을 사용하기도 한다.
- **3-Way Partitioning:** 중복된 키 값이 많은 경우, 배열을 '작은 값', '같은 값', '큰 값' 세 부분으로 나누면 성능을 개선할 수 있다.
